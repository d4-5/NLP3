**Напрям:** B) Extraction / NER  

**Інструмент:** Stanza (для лем, POS, токенізації)  

**Baseline:**  
- Baseline 1: регулярні вирази на original text (processed_v2)  
- Baseline 2: POS-патерни + леми через Stanza  

**Результати (порівняння):**  
- Baseline 1 (regex): знайшов більшість сутностей, Precision ≈ 1.0  
- Baseline 2 (POS/lemma): пропустив деякі нестандартні сутності (Precision ≈ 0.8)  

**Error Analysis (edge cases):**  
- Приклади помилок лематизації / POS:
  - `"Києві Кримінальне"` → не знайдено POS-baseline
  - Трансліт / латиниця (`Privit`) → POS ставить невірний UPOS
  - Дефіси / апострофи (`Марія-Анна`, `О'Ніл`) → regex пропускає деякі форми

**Висновок щодо лем/POS:**  
- Для поточного NER-завдання леми та POS **не покращили результат**  

**Рішення для проекту:**  
- **Не використовуємо леми/POS** у production extraction
